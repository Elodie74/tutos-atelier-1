Avant de commencer : c'est quoi GitHub ?

Imagine que tu travailles sur un document Word important. Tu le sauvegardes sur ton ordinateur, mais si ton PC plante, tu perds tout. Alors tu le mets sur Google Drive — comme ça, il est en sécurité en ligne et tu peux y accéder de partout.

GitHub, c'est exactement la même chose, mais pour du code.

C'est un site web (gratuit) où tu déposes tes fichiers de code. Ils sont stockés en ligne, en sécurité, et accessibles de partout. C'est aussi ce qui permet à Vercel de lire ton code et de le transformer en un vrai site web.

💡 L'analogie Google Drive
Google Drive → GitHub
Un dossier → Un "dépôt" (repository)
Un fichier → Un fichier (pareil !)
Enregistrer → Faire un "commit"

Tu n'as rien à installer sur ton ordinateur. Tout se fait depuis ton navigateur, directement sur le site github.com. Tu vas juste créer des fichiers et coller du code dedans — exactement comme si tu créais un Google Doc et que tu collais du texte.

🤗
Pas de panique : GitHub est utilisé par des millions de développeurs, mais tu n'as pas besoin d'être dev pour l'utiliser ! Pour nous, c'est juste un endroit où poser 4 fichiers. Rien de plus.

Créer un nouveau dépôt GitHub

On va créer un "dépôt" — c'est-à-dire un dossier en ligne qui va contenir les 4 fichiers de ton dashboard. Ça prend 2 minutes.

  • a
    Va sur github.com et connecte-toi à ton compte Si tu n'as pas encore de compte, clique sur "Sign up" et crée-en un gratuitement avec ton email.
  • b
    En haut à droite de l'écran, repère le petit bouton + C'est un petit "+" à côté de ta photo de profil. Clique dessus.
  • c
    Dans le menu qui s'ouvre, clique sur "New repository" "Repository" = dépôt en anglais. C'est le nom pour un dossier de projet sur GitHub.
+
▾ New repository

Le bouton "+" est tout en haut à droite, à côté de ta photo de profil

  • d
    Dans le champ "Repository name", tape : flowboard C'est le nom de ton projet. Tout en minuscules, pas d'espaces, pas d'accents.
  • e
    Laisse coché "Public" Ça veut dire que tout le monde peut voir ton code. C'est nécessaire pour que Vercel puisse y accéder. Pas d'inquiétude : personne ne va fouiller dedans !
  • f
    Coche la case "Add a README file" Très important ! Sans ça, GitHub crée un dépôt vide dans lequel tu ne pourras pas ajouter de fichiers depuis le navigateur.
  • g
    Clique sur le bouton vert Create repository
💡
Pourquoi le README est important ?
Le README crée une première "version" de ton dossier (on appelle ça une "branche" en langage GitHub). C'est grâce à ça que tu pourras ensuite ajouter d'autres fichiers directement depuis ton navigateur, sans aucun logiciel à installer.
🎉
Bravo ! Tu es maintenant sur la page de ton dépôt. Tu vois un fichier README.md — c'est normal, c'est celui que GitHub a créé automatiquement. Dans les prochaines étapes, on va ajouter les 4 fichiers de ton dashboard juste à côté.
